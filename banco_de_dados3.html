Modelagem de dados

O que é modelagem de Banco de Dados??

Processo de representar a estrutura lógica de um sistema de informação, criando um mapa visual de como os dados serão armazenados, organizados e relacionados. Basicamente, criar uma planta baixa para a estrutura do banco de dados.


  • Características de uma boa modelagem de Banco de Dados:
    1. Integridade:Dados corretos e consistentes (sem duplicações ou referências quebradas).
    2. Performance: Estrutura bem modelada, permite consultas rápidas e operações eficientes.
    3. Manutennbilidade: Facilita futuras modificações e expansões do sistema sem quebrar funcionalidades.

    Porque modelar antes de programar?

    Evita retrabalho, mais comunicação clara, consistência e qualidade.


    Modelo Conceitual de dados

    O que é?

    Desenho simples que mostra como as informações se organizam. Mostra o que guardar e não como.

  • Conceitos fundamentais do modelo conceitual:
    1. Entidade:

      • Coisas importantes do sistema.
      • Exemplos: Clientes, produtos, pedidos...

    2. Atributo:
      • Características de cada entidade.
      • Exemplo: Produto (entidade) contém: Nome, Preço, ID.

    3. Relacionamento:
      • Como as entidades se conectam
      • Exemplo:
        1. Cliente faz pedido.
        2. Pedido contém produto.

    4. Cardinalidade:
      • Define quantas instâncias de uma entidade se relacionam com outras.
      • Exemplo:
        • Um cliente pode fazer muitos pedidos (1:N).
        • Porém, um pedido não pode ter vários clientes.

    Modelo Entidade-Relacionamento (ER)
